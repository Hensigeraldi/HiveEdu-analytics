HIVEEDU PROJECT SHARING & LOCAL SETUP GUIDE
==========================================

Project Name:
HiveEdu Web-Based E-Report System Integrated with Learning Analytics

Stack:
- Frontend: Next.js
- Backend: NestJS
- Database: PostgreSQL with TypeORM
- Security: JWT & RBAC
- Roles: ADMIN, TEACHER, USER

Final MLR Formula:
Y = a + b1X1 + b2X2 + b3X3

Where:
X1 = attendance percentage
X2 = average tryout score
X3 = teacher objective score
Y = predicted next exam score


1. WHAT TO SHARE
================

Do not share the full raw project folder directly if it contains node_modules, .next, dist, or .env files.

Create a clean sharing folder, for example:

hiveedu-eraport-share/
├─ backend-api/
├─ frontend-web/
├─ database/
│  └─ hiveedu_eraport_dump.sql
└─ README_SETUP.txt

===============================

├─ src/

├─ tsconfig.json

├─ .env.example

├─ tsconfig.json

4. WHY NODE_MODULES SHOULD NOT BE SHARED
========================================

Do not send node_modules.

Reasons:
- node_modules can contain tens of thousands of files.
- Compressing node_modules makes ZIP creation very slow.
- The ZIP file becomes very large.
- Dependencies can be restored from package.json and package-lock.json.
- Your friend only needs to run npm install in backend-api and frontend-web.

As long as package.json and package-lock.json are included, your friend can restore all libraries.


5. HOW TO CREATE A CLEAN SHARE FOLDER
=====================================

Open CMD and run:

mkdir D:\hiveedu-share
mkdir D:\hiveedu-share\backend-api
mkdir D:\hiveedu-share\frontend-web
mkdir D:\hiveedu-share\database

Copy backend without node_modules and dist:

robocopy D:\hiveedu-eraport\backend-api D:\hiveedu-share\backend-api /E /XD node_modules dist /XF .env *.log

Copy frontend without node_modules and .next:

robocopy D:\hiveedu-eraport\frontend-web D:\hiveedu-share\frontend-web /E /XD node_modules .next out /XF .env.local *.log

Copy database dump:

copy D:\hiveedu_eraport_dump.sql D:\hiveedu-share\database\

Then compress D:\hiveedu-share into ZIP.


6. HOW TO EXPORT POSTGRESQL DATABASE
====================================

If pg_dump is recognized, run:

pg_dump -U postgres -h localhost -p 5432 -d hiveedu_eraport -f D:\hiveedu_eraport_dump.sql

If pg_dump is not recognized, use the full PostgreSQL path.

For PostgreSQL 16:

"C:\Program Files\PostgreSQL\16\bin\pg_dump.exe" -U postgres -h localhost -p 5432 -d hiveedu_eraport -f D:\hiveedu_eraport_dump.sql

For PostgreSQL 17:

"C:\Program Files\PostgreSQL\17\bin\pg_dump.exe" -U postgres -h localhost -p 5432 -d hiveedu_eraport -f D:\hiveedu_eraport_dump.sql

After export, check that the file exists:

dir D:\hiveedu_eraport_dump.sql

If the file exists, copy it into:

D:\hiveedu-share\database\hiveedu_eraport_dump.sql


7. BACKEND ENV EXAMPLE

9. SETUP GUIDE FOR YOUR FRIEND
==============================

After your friend receives the ZIP, they should:

1. Extract ZIP.

2. Install requirements:
- Node.js 20 or newer
- PostgreSQL 16 or newer
- npm

3. Create PostgreSQL database:

createdb -U postgres -h localhost -p 5432 hiveedu_eraport

If createdb is not recognized:

"C:\Program Files\PostgreSQL\16\bin\createdb.exe" -U postgres -h localhost -p 5432 hiveedu_eraport

4. Import database dump:

psql -U postgres -h localhost -p 5432 -d hiveedu_eraport -f database\hiveedu_eraport_dump.sql

If psql is not recognized:

"C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -h localhost -p 5432 -d hiveedu_eraport -f database\hiveedu_eraport_dump.sql

13. OPTIONAL: RUN SEEDER INSTEAD OF DATABASE DUMP

14. PORT RULES
==============

Use these ports:

Backend NestJS:
http://localhost:3000

Frontend Next.js:
http://localhost:3001

Do not run frontend on port 3000 because backend uses port 3000.

If port 3000 is already in use, check:

netstat -ano | findstr :3000

Find the PID on the LISTENING line, then kill it:

taskkill /PID PID_NUMBER /F

Example:

taskkill /PID 18000 /F

Then run backend again:

cd /d D:\hiveedu-eraport\backend-api
npm run start:dev


15. COMMON ERRORS AND FIXES
===========================

Error:
EADDRINUSE: address already in use :::3000

Meaning:
Port 3000 is already used by another process.

Fix:

netstat -ano | findstr :3000
taskkill /PID PID_NUMBER /F

Then run backend again.


Error:
pg_dump is not recognized

Meaning:
PostgreSQL bin folder is not in PATH.

Fix:
Use full path:

"C:\Program Files\PostgreSQL\16\bin\pg_dump.exe" -U postgres -h localhost -p 5432 -d hiveedu_eraport -f D:\hiveedu_eraport_dump.sql


Error:
psql is not recognized

Fix:
Use full path:

"C:\Program Files\PostgreSQL\16\bin\psql.exe" -U postgres -h localhost -p 5432 -d hiveedu_eraport -f database\hiveedu_eraport_dump.sql


Error:
Invalid credentials

Possible causes:
- Database was not imported.
- Seeder was not run.
- Backend points to wrong database.
- .env DB password is wrong.

Fix:
Check backend-api/.env:

DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=your_postgres_password
DB_DATABASE=hiveedu_eraport

Then restart backend.


Error:
Frontend cannot reach backend

Fix:
Check frontend-web/.env.local:

NEXT_PUBLIC_API_URL=http://localhost:3000

Then restart frontend.
